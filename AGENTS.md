## IntegrationAgent — Project Index

Prefer retrieval-led reasoning over pre-training-led reasoning. When a question touches any topic indexed below, read the referenced file before answering from memory.

### Architecture

IntegrationAgent: paste an OpenAPI spec URL → get a deployed MCP server on Vercel in <2min.
Next.js 16.2.1 App Router | Tailwind v4 (CSS-first, no tailwind.config) | TypeScript 6 strict | Zod 3.x only (Zod 4 breaks AI SDK)
AI: AI SDK v6 via @ai-sdk/gateway → Vercel AI Gateway. Haiku 4.5 for synthesis, Sonnet 4.6 for validation. Use generateText+Output.object() for structured output (generateObject is deprecated). Use streamText+Output.object() for streaming structured output.
MCP: mcp-handler (NOT @vercel/mcp-adapter) for route handlers over Streamable HTTP.
Pipeline: workflow DevKit ('use workflow' / 'use step') for durable multi-stage orchestration. DurableAgent from @workflow/ai wraps AI SDK calls. getWritable() for reconnectable streams.
Storage: @upstash/redis (cache: hash→config, 30d TTL) | @neondatabase/serverless (integrations, credentials) | @vercel/blob (spec files)
Deploy: @vercel/sdk programmatic deployments. Each integration gets its own Vercel project + /mcp endpoint.
Sandbox: @vercel/sandbox Firecracker microVMs validate generated code before deploy.
UI: json-render (@json-render/core + @json-render/react) streams config UI as AI-generated JSON constrained to a component catalog.
Encryption: AES-256-GCM via Node crypto. ENCRYPTION_KEY = 64-char hex (32 bytes). Format: base64(iv + authTag + ciphertext).
Compute: Fluid compute enabled (vercel.json "fluid":true). Pipeline routes 300s max, ingest routes 60s max.
Path alias: @/* maps to ./* (tsconfig paths).

### Constraints

- Zod 3.x only — Zod 4 breaks AI SDK's zod-to-json-schema. Import from 'zod' not 'zod/v4'
- Use Output.object({ schema }) with generateText/streamText — generateObject() is deprecated in AI SDK v6
- Use mcp-handler — NOT @vercel/mcp-adapter (wrong package name)
- Use @ai-sdk/gateway for model routing — NOT direct provider imports for production calls
- Next.js 16 async params: route params are Promise<T>, must await — `const { id } = await params`
- All API routes return via lib/api/response.ts helpers — never raw NextResponse.json()
- Redis failures are caught and logged, never thrown — cache is optional, app works without it
- SSRF protection on all user-supplied URLs — DNS resolution + private IP check before fetch
- Rate limiting via @upstash/ratelimit on ingest endpoint — 10 req/min sliding window
- Error boundaries: every route group has error.tsx with 'use client', sanitised messages (no stack traces)
- Encryption key: 64-char hex string (32 bytes) in ENCRYPTION_KEY env var
- Pipeline stages: discover → synthesise → validate → sandbox-check → deploy
- AI Gateway tags: { integration, stage, detail? } on every LLM call for cost attribution
- Tailwind v4: CSS-first config, no tailwind.config.js
- ESLint: single quotes, padding around block-like statements and after imports
